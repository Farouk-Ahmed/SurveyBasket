## المرحلة 1: إصلاح 500 في Auth (Register + Login)

**الهدف:** TC001, TC002, TC004, TC005, TC007, TC008, TC009 تنجح بعد إصلاح Auth.

### 1.1 التأكد من تطبيق التعديلات السابقة وإعادة تشغيل الـ API

- [x] التأكد أن `AuthService` يستخدم `_dbContext.Users.Include(u => u.RefreshTokens)` ثم `SaveChangesAsync` لحفظ الـ RefreshToken (مطبقة).
- [x] إضافة try-catch حول `SendWelcomeEmailAsync` حتى لا يفشل التسجيل إذا فشل الإيميل.
- [x] معالجة null في `EnsureAccountHasRoleAsync` (FirstName, LastName).
- [ ] إيقاف الـ API الحالي ثم تشغيله من جديد حتى يُحمَّل الكود الأخير:
  ```powershell
  dotnet run --project SurveyBasket.API\SurveyBasket.API.csproj
  ```

### 1.2 إذا استمر 500: تحديد الاستثناء الفعلي

- [ ] إضافة معالجة استثناء في `AuthController` للـ Login و Register بحيث يُعاد في جسم الـ 500 رسالة الاستثناء الداخلية (مؤقتاً للتشخيص)، أو
- [ ] تفعيل logging للاستثناءات ومراجعة السجلات عند استدعاء Login/Register.
- [ ] مراجعة:
  - `EnsureAccountHasRoleAsync` (إنشاء/تحديث Account).
  - `GetProfilePicturePathAsync` (استعلام Account).
  - `_emailService.SendWelcomeEmailAsync` في Register (إن كان يعمل بشكل متزامن ويُرمى منه استثناء).

### 1.3 التحقق من البيئة

- [ ] التأكد من تشغيل الـ Database ونجاح الاتصال.
- [ ] التأكد من وجود مستخدم الـ Seed (مثل admin@surveybasket.com / Admin@123) للاختبارات التي تعتمد على Login.

## المرحلة 2: إصلاح 500 في GET /api/Polls (TC010)

**الهدف:** TC010 ينجح بدون مصادقة.

### 2.1 تحديد سبب الاستثناء

- [ ] مراجعة `PollsController.GetPolls` و `PollService.GetPollsAsync`.
- [ ] التأكد من:
  - عدم استثناء عند عدم وجود مستخدم مصادق (الـ endpoint مسموح له بـ `[AllowAnonymous]`).
  - عدم وجود null reference عند بناء الـ response (مثل `Attachments` أو `UploadedBy` أو `Request`).

### 2.2 إصلاح الكود

- [x] معالجة الحالات التي قد تكون null (مثلاً قوائم مرفقات، مستخدم رافع الملف، أو عناوين الملفات) في `PollsController.GetPolls`.
- [x] عند عدم مصادقة المستخدم (AllowAnonymous) إرجاع كل الاستطلاعات غير المحذوفة بدلاً من تصفية `CreatedById == null` في `PollService.GetPollsAsync`.

## المرحلة 4: التحقق من توافق الـ API مع توقعات الاختبارات

بعد إصلاح 500، التأكد من:

- [ ] **Register:** يرجع **201 Created** وجسم يحتوي على **`id`** (مثل `{ "id": "..." }`).
- [ ] **Login:** يرجع **200 OK** وجسم يحتوي على **`token`** و **`refreshToken`** (وأي أسماء بديلة مستخدمة في الـ API مثل `jwtToken`/`jwt` إن وُجدت).
- [ ] **Refresh:** يقبل body فيه **`refreshToken`** فقط ويرجع **200** مع **`token`** و **`refreshToken`** جديد.
- [ ] **Clients/me:** يتطلب مصادقة ويرجع 200 مع بيانات الملف الشخصي، و 401 بدون مصادقة.
- [ ] **Dashboard/create-user:** يرجع 201 للمشرف و 403 لغير المشرف عند المحاولة.
- [ ] **Dashboard/summary و audit-logs:** يرجعان 200 مع JWT صالح.
